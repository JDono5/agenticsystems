# core/error_handler.py  import time import requests  def api_call_with_retry(fn, max_retries=3, agent_name="unknown"):     for attempt in range(max_retries):         try:             return fn()         except Exception as e:             wait = (2 ** attempt)          # 1s, 2s, 4s backoff             if attempt < max_retries - 1:                 time.sleep(wait)             else:                 log_error(agent_name, str(e))                 send_telegram_alert(                     f"⚠️ {agent_name} failed after {max_retries} attempts: {e}"                 )                 return None               # skip this task, don't crash

Failure Type | Behavior
Etsy API rate limit (429) | Retry with backoff. If still failing after 3 attempts, skip listing, alert via Telegram.
Printify upload fails | Retry 3 times. If fails, design stays as 'generated' status, will retry next run.
GPT-4o image gen fails | Retry 3 times. If fails, log and skip. Research brief stays in Supabase for next day.
Playwright scrape blocked | Fallback to manual niche input stored in .env. Alert sent immediately.
Spend cap hit | All agents halt. Telegram alert sent. System waits until next calendar month.
Railway process crash | Railway auto-restarts. APScheduler resumes on restart. No data loss via Supabase.

# core/spend_monitor.py  from core.supabase_client import supabase from datetime import datetime  MONTHLY_CAP = float(os.getenv("MONTHLY_SPEND_CAP", 100))  def check_cap():     start_of_month = datetime.now().replace(day=1, hour=0, minute=0)     result = supabase.table("cost_log")\         .select("cost_usd")\         .gte("timestamp", start_of_month.isoformat())\         .execute()          total_spent = sum(r["cost_usd"] for r in result.data)          if total_spent >= MONTHLY_CAP:         send_telegram_alert(             f"STOP: Monthly spend cap of $[MONTHLY_CAP] hit. "             f"All agents paused until next month."         )         return False     return True

The most common failure mode is not a technical problem — it's killing the system at week 3 because results are slower than expected. Etsy takes 4-8 weeks to build organic traction. This is normal.

Rule | Why It Matters
Build in order. Don't skip phases. | Each phase depends on the previous. Skipping ahead creates debugging hell.
Day 1 goal only: one live listing. | Proves Etsy + Printify integration works before you build anything else on top of it.
One niche for the first 30 days. | Diffusing across multiple niches before one is working dilutes signal. You won't know what's working.
Don't auto-publish for 2 weeks. | New shops with no history get flagged faster. Manual review protects the account early.
Set the spend cap before anything runs. | Without it a bug could run thousands of API calls overnight.
Give it 60 days before any judgment. | Month 1 is ramp-up. Month 2 is when real data appears. Killing it at week 3 is the #1 mistake.
Don't tinker with it daily. | Constant prompt changes and niche switches prevent you from seeing what's actually working.
Fix errors immediately when alerted. | A broken research agent running silently for a week generates junk designs and wasted API cost.
