1. Vision & Core Principles
An autonomous multi-stream income engine. AI agents research demand, generate assets, publish to platforms, optimize performance, and find new income streams. The owner approves proposals and provides credentials when needed. Everything else runs itself.
Two rules above all others: 1. Get money moving before adding complexity. Publisher before orchestrator. Revenue before self-expansion. 2. The system never asks the owner for something it could do itself. Only credentials and strategic approvals require human input.

1.1 Owner Interaction Model
The owner interacts with the system in exactly three ways:
Weekly dashboard review (~15 min). Read the report. Review Scout proposals. Approve or ignore.
Approve a new income stream when Scout proposes one. One tap in the dashboard.
Provide a credential when asked. The orchestrator gives exact steps. Owner does that one action. System continues automatically.
Example credential request from orchestrator: 'To launch Pinterest I need your API key. Go to pinterest.com/business -> Developers -> Create App -> copy the access token. Paste it here and I will handle the rest.'

1.2 The Three Layers
Layer | Who | Does What
1 - Owner | You | Approves proposals. Provides credentials. Reviews weekly report. Nothing else.
2 - Orchestrator | orchestrator_agent.py | Receives owner messages. Manages all workers. Routes tasks. Monitors system. Surfaces proposals.
3 - Workers | All other agents | Execute specific jobs. Never interact with owner directly. All communication routes through orchestrator.

CREATE TABLE IF NOT EXISTS research_briefs (   id                          uuid PRIMARY KEY DEFAULT gen_random_uuid(),   created_at                  timestamptz NOT NULL DEFAULT now(),   platform                    text NOT NULL DEFAULT 'etsy',   niche                       text NOT NULL,   sub_niche                   text,   top_keywords                text[] NOT NULL DEFAULT '{}',   opportunity_summary         text NOT NULL,   recommended_design_direction text,   recommended_price           numeric(10,2),   avg_competitor_price        numeric(10,2),   competitor_title_examples   text[] DEFAULT '{}',   raw_scrape_data             jsonb,   tokens_used                 integer DEFAULT 0,   cost_usd                    numeric(10,6) DEFAULT 0 ); CREATE INDEX idx_briefs_platform ON research_briefs(platform); CREATE INDEX idx_briefs_created ON research_briefs(created_at DESC);

3.2 designs
CREATE TABLE IF NOT EXISTS designs (   id                uuid PRIMARY KEY DEFAULT gen_random_uuid(),   created_at        timestamptz NOT NULL DEFAULT now(),   brief_id          uuid REFERENCES research_briefs(id) ON DELETE SET NULL,   platform          text NOT NULL DEFAULT 'etsy',   niche             text NOT NULL,   file_path         text,   prompt_used       text,   variation_angle   text,   generation_cost   numeric(10,6) DEFAULT 0,   qa_cost           numeric(10,6) DEFAULT 0,   qa_reason         text,   status            text NOT NULL DEFAULT 'generated'                     CHECK (status IN ('generated','approved','rejected','published')),   attempts          integer DEFAULT 1 ); CREATE INDEX idx_designs_status ON designs(status); CREATE INDEX idx_designs_platform ON designs(platform); CREATE INDEX idx_designs_created ON designs(created_at DESC);

CREATE TABLE IF NOT EXISTS listings (   id                    uuid PRIMARY KEY DEFAULT gen_random_uuid(),   created_at            timestamptz NOT NULL DEFAULT now(),   design_id             uuid REFERENCES designs(id) ON DELETE SET NULL,   platform              text NOT NULL,   external_listing_id   text,   printify_product_id   text,   title                 text NOT NULL,   description           text,   tags                  text[] DEFAULT '{}',   price                 numeric(10,2),   status                text NOT NULL DEFAULT 'draft'                         CHECK (status IN ('draft','active','paused','removed')),   published_at          timestamptz,   impressions           integer DEFAULT 0,   clicks                integer DEFAULT 0,   last_optimized_at     timestamptz,   niche                 text ); CREATE INDEX idx_listings_platform ON listings(platform); CREATE INDEX idx_listings_status ON listings(status);

CREATE TABLE IF NOT EXISTS cost_log (   id            uuid PRIMARY KEY DEFAULT gen_random_uuid(),   timestamp     timestamptz NOT NULL DEFAULT now(),   agent         text NOT NULL,   provider      text NOT NULL CHECK (provider IN ('anthropic','openai','google','sendgrid','other')),   model         text NOT NULL,   tokens_used   integer NOT NULL DEFAULT 0,   cost_usd      numeric(10,6) NOT NULL DEFAULT 0,   notes         text ); CREATE INDEX idx_cost_timestamp ON cost_log(timestamp DESC); CREATE INDEX idx_cost_agent ON cost_log(agent);

3.5 sales
CREATE TABLE IF NOT EXISTS sales (   id                uuid PRIMARY KEY DEFAULT gen_random_uuid(),   order_date        timestamptz NOT NULL DEFAULT now(),   platform          text NOT NULL,   external_order_id text NOT NULL,   listing_id        uuid REFERENCES listings(id) ON DELETE SET NULL,   gross_revenue     numeric(10,2) NOT NULL,   printify_cost     numeric(10,2) NOT NULL DEFAULT 0,   platform_fee_pct  numeric(5,4) NOT NULL DEFAULT 0.065,   platform_fee_flat numeric(10,2) NOT NULL DEFAULT 0.20,   net_profit        numeric(10,2) GENERATED ALWAYS AS                     (gross_revenue - printify_cost                      - (gross_revenue * platform_fee_pct)                      - platform_fee_flat) STORED ); CREATE INDEX idx_sales_platform ON sales(platform); CREATE INDEX idx_sales_date ON sales(order_date DESC);

14. Rules for Not Failing
Rule | Why
Build in strict day order. | Every phase depends on the previous. Skipping creates untraceable failures.
Revenue before complexity. | Publisher before orchestrator. Prove the loop works before adding intelligence.
One new stream at a time. | Parallel launches make it impossible to know what is working.
Draft mode 2 weeks on every new platform. | New accounts with no history get flagged faster. Protects account health.
$100 cap configured before anything runs. | One bug without a cap can run thousands of API calls overnight.
60 days before judging any stream. | Month 1 is ramp-up. Month 2 is real signal. Week 3 is too early to decide anything.
Scout proposals need one approval. | System never spends on new streams the owner has not seen.
Orchestrator cannot exceed cap without approval. | Self-expansion is bounded. The system grows within owner-set limits.
Log every expense from day one. | All-time P&L only means something if every cost including the $29 fee is captured.
Never commit .env to GitHub. | One push exposes all API keys. .env must be in .gitignore before first commit.
Owner only does credentials and approvals. | If the system is asking for anything else it is a design flaw to fix, not a task to do.
